The Babel plugin (`react-native-boost/plugin`) is the core of React Native Boost.

Defaults are safe and usable out of the box, but you can tune behavior for your app or monorepo.

## Example Configuration

```js
// babel.config.js
module.exports = {
  plugins: [
    [
      'react-native-boost/plugin',
      {
        verbose: false,
        silent: false,
        ignores: ['node_modules/**'],
        optimizations: {
          text: true,
          view: true,
        },
      },
    ],
  ],
};
```

## Practical Notes

### Logging behavior

- `silent` disables all logs.
- With `silent: false`, optimized component logs are shown by default.
- `verbose` adds skip-reason logs.

### Monorepo `ignores`

Ignore patterns are resolved from Babel's current working directory.

<Callout title="Monorepo Note">
  In nested apps, use explicit parent paths when needed, for example `../../node_modules/**`.
</Callout>

### Dangerous view optimization

`dangerouslyOptimizeViewWithUnknownAncestors` can increase optimization coverage, but it can also introduce regressions.

<Callout type="warn" title="Use cautiously">
  Keep this disabled unless you have verified your component tree behavior and tested affected screens carefully.
</Callout>

## Environment-Specific Enablement

You can enable React Native Boost by environment with Babel `env` config:

```js
module.exports = {
  env: {
    development: {
      plugins: ['react-native-boost/plugin'],
    },
  },
};
```

See Babel docs: https://babeljs.io/docs/options#env
